# AGENTS.md

## Build / Test / Lint Commands

```bash
make build-all          # Build all Go services to bin/
make build-inventory    # Build Rust inventory service (cargo build --release)
make test               # Run all tests (Go + Rust with --test-threads=1)
make lint               # Run golangci-lint, cargo clippy, ruff check + format check
make gen                # Regenerate all code (proto ‚Üí Go, proto ‚Üí Rust, sqlc, OpenAPI docs)
```

Single service:
```bash
cd services/<service> && go test ./...
cd services/<service> && go test ./... -run TestFunctionName
cd services/<service> && golangci-lint run
cd services/inventory-service && cargo test -- --test-threads=1
cd services/analytics-worker && uv run pytest
```

Verification order before committing: `make gen` ‚Üí `make build-all` ‚Üí `make lint` ‚Üí `make test`

Note: `make up` (docker compose up) runs `make gen` first as a prerequisite.

```bash
make test-integration    # Run integration tests (starts docker-compose, waits for health, runs gateway E2E)
make install-git-hooks   # Install pre-commit hook that auto-runs codegen when .proto files are staged
```

## Architecture

Spec-first polyglot microservices monorepo. Proto files in `proto/` are the source of truth.

| Service | Language | gRPC Port | Metrics Port |
|---------|----------|-----------|--------------|
| gateway | Go | ‚Äî | 8080 (HTTP) |
| product-service | Go | 9091 | 8091 |
| order-service | Go | 9092 | 8092 |
| user-service | Go | 9103 | 8103 |
| payment-service | Go | 9104 | 8104 |
| inventory-service | Rust | 9105 | 8105 |
| delivery-service | Go | 9106 | 8106 |
| analytics-worker | Python | ‚Äî | ‚Äî |

Go 1.24 workspace mode (`go.work`). Each Go service has its own `go.mod`. Services import generated proto code via `github.com/afasari/shinkansen-commerce/gen/proto/go/<service>`.

Internal communication: gRPC. External: REST via Gateway (port 8080). Events: Kafka. Data: PostgreSQL + Redis.

## Code Generation (CRITICAL)

### Proto changes
1. Edit `.proto` in `proto/<service>/`
2. `make proto-gen` ‚Üí generates Go gRPC code to `gen/proto/go/`
3. `make proto-gen-rust` ‚Üí generates Rust proto code to `gen/proto/rust/`
4. `make proto-openapi-gen` ‚Üí generates OpenAPI docs to `services/gateway/docs/api/`

### SQL changes (Go services only)
1. Add/edit `.sql` files in `services/<service>/internal/queries/` with `-- name: FunctionName :mode` annotation
2. `make sqlc-gen` ‚Üí generates typed Go code to `services/<service>/internal/db/`

**`make sqlc-gen` only processes product-service and order-service.** Payment, user, and delivery services have their sqlc steps commented out in the Makefile.

**NEVER edit generated code** (files with `// Code generated by ... DO NOT EDIT.` header). The `gen/` directory and `internal/db/` directories are generated.

## Service Layout

Go services follow: handler ‚Üí service ‚Üí repository ‚Üí db (sqlc) ‚Üí PostgreSQL

```
services/<go-service>/
‚îú‚îÄ‚îÄ cmd/<service>/main.go
‚îú‚îÄ‚îÄ internal/
‚îÇ   ‚îú‚îÄ‚îÄ handler/         # gRPC handlers
‚îÇ   ‚îú‚îÄ‚îÄ service/         # Business logic
‚îÇ   ‚îú‚îÄ‚îÄ repository/      # Data access
‚îÇ   ‚îú‚îÄ‚îÄ db/              # sqlc-generated (DO NOT EDIT)
‚îÇ   ‚îú‚îÄ‚îÄ queries/         # SQL queries for sqlc
‚îÇ   ‚îú‚îÄ‚îÄ migrations/      # Database migrations (XXXXXXX_description.{up,down}.sql)
‚îÇ   ‚îú‚îÄ‚îÄ cache/           # Redis caching
‚îÇ   ‚îú‚îÄ‚îÄ config/          # Env-based config loading
‚îÇ   ‚îî‚îÄ‚îÄ pkg/             # Internal utilities
‚îú‚îÄ‚îÄ sqlc.yaml
‚îî‚îÄ‚îÄ go.mod
```

Gateway is different ‚Äî no DB/sqlc layer. Structure: `cmd/` ‚Üí `internal/handler/` + `internal/client/` + `internal/middleware/` + `internal/cache/`. It proxies REST‚ÜîgRPC rather than owning data.

Rust service (inventory-service): `src/main.rs` ‚Üí `service.rs` ‚Üí `repository.rs` (sqlx with compile-time checked macros). Proto codegen lives in `gen/proto/rust/build.rs` (a separate Cargo crate `shinkansen-proto`); the inventory-service's own `build.rs` is a 3-line stub that just triggers reruns. Migrations are at `services/inventory-service/migrations/` (root, not internal/) ‚Äî single SQL files, not up/down pairs.

Python service (analytics-worker): `uv`-managed, entry point `analytics_worker/cli.py`.

## Database

Each service owns a PostgreSQL schema (e.g., `catalog`, `orders`, `users`, `payments`, `inventory`, `delivery`). Single database `shinkansen`.

```bash
make db-migrate                          # All services
make db-rollback                         # Rollback last migration per service
cd services/<service> && migrate -path internal/migrations -database "$DATABASE_URL" up
```

Default local DATABASE_URL: `postgres://shinkansen:shinkansen_dev_password@localhost:5432/shinkansen?sslmode=disable`

## Integration Tests

Integration tests live at `services/<service>/test/integration/` (outside `internal/`). Run via:

```bash
make test-integration    # Calls scripts/run-integration-tests.sh (starts docker-compose, waits for health, runs gateway E2E)
```

## Go Conventions

- Import groups: stdlib ‚Üí external (alpha) ‚Üí internal (alpha)
- Error wrapping: `fmt.Errorf("failed to X: %w", err)`
- Structured logging: `go.uber.org/zap`
- Testing: table-driven with `t.Run()`, `github.com/stretchr/testify/assert` + `require`
- Redis mocking in tests: `github.com/alicebob/miniredis/v2`
- Config via env vars with defaults using `getEnv()` in `internal/config/config.go`
- UUID conversion: use `pgutil.ToPG()` / `pgutil.FromPG()` ‚Äî copy-pasted per-service at `internal/pkg/pgutil/uuid.go` (not a shared import)
- sqlc driver: `pgx/v5`

## Infrastructure

`docker-compose.yml` starts PostgreSQL 15, Redis 7, and all services. No Kafka/MinIO/Jaeger/Prometheus/Grafana in the compose file (despite README mentioning them).

`deploy/k8s/base/k8s.yaml` has real Deployment, Service, ConfigMap, Secret, and HPA manifests for production use. `deploy/terraform/` and `deploy/docker/` are empty.

**Warning:** `deploy/prometheus/prometheus.yml` has stale/incorrect scrape ports for every service ‚Äî do not trust it as a source of truth for metrics endpoints.

Observability endpoints:
- Grafana: `http://localhost:3000` (admin/admin) ‚Äî if deployed separately
- Jaeger: `http://localhost:16686` ‚Äî if deployed separately

## CI

`.github/workflows/ci-cd.yml` ‚Äî Only the **lint** job is active. Test and build jobs are commented out. CI runs: `make proto-gen` ‚Üí `make sqlc-gen` ‚Üí `make lint` ‚Üí proto format check. Go 1.24, golangci-lint v2.9.0.
